Trading features

Description

Tokemon provides powerful trading options for ERC20 tokens, which you can leverage to maximize your profits. You can configure various different strategies for a trading per pair - the token you want to trade and the token you want to trade against. The trading is performed by a two-part architecture - frontend and backend. The frontend is an automated script which you run on your machine, and the backend is a trading bot which messages you (the frontend script) when a trade should happen. Your script then executes the command from your machine. No private keys are shared in any step, and the architecture is created in this way for this purpose mainly. The other advantage of this implementation is that the client is very lightweigh, it only listens for trading signals and executes transactions. The client is currently a script, but our plan is to make this a packaged multi-platform application which will be more user friendly. The architecture diagram is shown below.

Trading bot configuration

The following are some of the configuration options for the trading bot:

  • Trading account - the account you want to trade with
  • Token in - the token you want to trade
  • Token out - the token you want to trade against
  • Max gas (gwei) - the maximum gas allowance for the trading transactions
  • Max slippage - the maximum amount of slippage allowed
  • Strategy specific inputs - different for different strategies, such as max tradeable amount, strategy threshold, etc.

Accounts

The trading configurations are specified per account. Accounts are simply a more meaningful/simpler way to use your ethereum addresses on the system. The accounts are created by name and address. Depending on the plan you choose, you can have unlimited number of accounts and you can trade with as many accounts as you want at the same time.
